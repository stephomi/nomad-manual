# ![](/icons/open.webp) 파일

파일 메뉴를 사용하면 Nomad 프로젝트를 저장·불러오기 하고, 3D 모델을 가져오거나 내보내며, 렌더된 이미지를 내보낼 수 있습니다.

![](/images/file_menu.webp)

## 프로젝트
![](/images/file_project.webp)

이 메뉴 상단에는 마지막 저장본의 썸네일이 표시됩니다. 이 썸네일을 누르면 미니 브라우저가 열리며, 다른 프로젝트를 두 번 탭하면 열기, 추가, 저장, 복제, 이름 바꾸기, 삭제를 할 수 있는 미니 메뉴가 나타납니다.

### ![](/icons/nomad.webp) 프리셋 
데모와 캐릭터 구성 요소 모음에 접근합니다. 하나를 선택한 뒤 다시 선택하면, 프로젝트 폴더로 항목을 열기, 씬에 추가, 복제 중에서 선택할 수 있습니다.

![](/images/file_preset_preview.webp)

### ![](/icons/save.webp) 저장
Nomad 프로젝트를 저장합니다.

### ![](/icons/save_as.webp) 다른 이름으로 저장...
프로젝트 브라우저를 표시하여 Nomad 프로젝트를 새 이름으로 저장할 수 있게 합니다.

### ![](/icons/pencil.webp) 이름 바꾸기
현재 프로젝트의 이름을 바꾸기 위한 텍스트 상자를 표시합니다.

### ![](/icons/open.webp) 열기...
프로젝트 브라우저를 표시하여 프로젝트를 엽니다.

### ![](/icons/add_file.webp) 씬에 추가...
프로젝트 브라우저를 표시하며, 프로젝트를 선택하면 그 내용이 현재 씬과 병합됩니다.

### ![](/icons/trash.webp) 삭제...
프로젝트 브라우저를 표시하며, 선택된 프로젝트는 파일 시스템에서 삭제됩니다.

### ![](/icons/new_file.webp) 새로 만들기
새 프로젝트를 시작합니다. 저장되지 않은 변경 사항이 있으면 저장 여부를 묻습니다.

### ![](/icons/autosave.webp) 자동 저장...
자동 저장 옵션을 제어하는 메뉴입니다.

자동 저장을 활성화하면, 일정 간격으로 팝업이 나타나도록 타이머를 설정할 수 있습니다.  
Nomad가 백그라운드에서 저장하지 않는 이유는 3D 파일이 상당히 클 수 있어 조각 작업 중에 눈에 띄는 지연을 유발할 수 있기 때문입니다.

또한 메모리 부족 문제를 피하기 위해, 저장 작업 전에 씬을 보통 압축합니다.  
이 압축/압축 해제 과정도 저장 작업을 느리게 만듭니다.

### 타이머 팝업
타이머 팝업이 얼마나 자주 나타날지 설정합니다.

### 팝업 제한 시간
팝업 제한 시간을 활성화합니다.

### 자동 저장 삭제
프로젝트에 대한 자동 저장 파일이 존재하면, 원본 프로젝트 대신 자동 저장 파일이 자동으로 로드됩니다. 이것이 필요하지 않다면, 이 버튼으로 자동 저장 파일을 삭제할 수 있습니다. 이후에는 마지막 수동 저장본이 로드됩니다.


## 가져오기

### ![](/icons/add_file.webp) 가져오기
Nomad 프로젝트가 아닌 3D 파일을 가져올 때 사용합니다.

외부 씬 파일을 Nomad로 가져올 때는 *가져오기(import)* 또는 *추가(add)* 중에서 선택할 수 있습니다.

파일을 추가하면 객체들이 현재 씬에 단순히 추가됩니다.  
파일을 가져오면 새 객체들로 새로운 Nomad 프로젝트가 생성됩니다.

Nomad가 가져올 수 있는 형식은 다음과 같습니다:
- Nomad (.nom)
- glTF (.glb, .gltf)
- OBJ (.obj)
- STL (.stl)
- PLY (.ply)
- FBX (.fbx, 실험적)

### ![](/icons/cog.webp) 고급
고급 가져오기 옵션을 표시합니다:

### Project/ glTF / OBJ / STL / FBX
#### 토폴로지 유지
Nomad는 기본적으로 로드 시 문제 있는 지오메트리를 수정하려고 시도합니다. 이 옵션을 활성화하면 정점/면 재정렬, 정점/면 중복 제거, 사용되지 않는 정점 제거를 하지 않습니다.

#### 텍스처 건너뛰기
glTF처럼 텍스처를 지원하는 형식에서 텍스처 로드를 건너뜁니다.

### Project / glTF
#### GUI 설정 유지
Nomad .nom 또는 glTF 파일 안에 GUI 및 프로젝트 설정을 저장하도록 활성화합니다.

### OBJ
#### 그룹별 OBJ 분할
OBJ 그룹을 개별 객체로 분할하여 가져오도록 활성화합니다.

#### 색 공간
OBJ에서 해석되는 색 모드를 Linear, sRGB, Auto 중에서 설정합니다.

### PLY
#### 색 공간
PLY에서 해석되는 색 모드를 Linear, sRGB, Auto 중에서 설정합니다.


### FBX
#### 색 공간
OBJ에서 해석되는 색 모드를 Linear, sRGB, Auto 중에서 설정합니다.

### 전체/보이는 것/선택된 것
활성화된 버튼 상태에 따라 어떤 객체들이 내보내질지 결정됩니다. 아이콘 옆 숫자는 해당 옵션으로 내보내질 객체 수를 나타냅니다.

### 버텍스 색상
파일 형식이 지원하는 경우 버텍스 색상을 내보냅니다.

### PBR 페인트
PBR 버텍스 색상은 보조 버텍스 색상 속성으로 내보내집니다.  
채널은 다음과 같이 패킹됩니다:

|           | Channel  |
| :-------: | :------: |
| Roughness | R        |
| Metalness | G        |
| Masking   | B        |


### 레이어
레이어는 glTF의 morph target을 통해 지원됩니다.  
Nomad는 레이어별 색상, 거칠기(roughness), 금속성(metalness)도 내보내지만, 다른 소프트웨어에서는 무시됩니다.

### 레이어 페인팅
레이어 페인팅을 내보냅니다. 보통 다른 소프트웨어에서는 무시됩니다.

### 페이스 그룹
페이스 그룹을 내보냅니다. 내보내기가 다른 소프트웨어와 충돌을 일으키는 경우가 있을 수 있습니다.

### 노멀
노멀 정보를 내보냅니다. Nomad는 다른 파일 형식을 가져올 때 항상 자체적으로 노멀을 다시 계산한다는 점에 유의하세요.

### 탄젠트
탄젠트 정보를 내보냅니다. 노멀 맵이 있는 모델에서 사용됩니다. 

### 텍스처
머티리얼에 텍스처가 추가되어 있다면 내보내집니다. 이 기능은 베이크를 수행하지 않으며, 베이크는 토폴로지의 베이크 옵션을 통해 수행됩니다.

### 내보내기 버튼
이 버튼을 클릭하면 선택된 설정으로 지오메트리를 내보냅니다.

::: tip Tip: Blender로 거칠기와 금속성 가져오기

Blender는 glTF/glb를 가져올 수 있지만, 금속성과 거칠기에 대한 버텍스 속성을 자동으로 인식하지는 않습니다. 이를 사용하려면 머티리얼 에디터에서 Vertex Color 노드를 만들고, 속성을 다음 색상 속성(보통 Col.001)으로 설정합니다. 그런 다음 'Separate XYZ' 노드를 연결하고, X를 Roughness에, Y를 Metallic에 연결합니다.

이 영상에서 과정을 볼 수 있습니다:

![](/videos/blender_pbr.mp4)

::: 

::: tip Tip: USD 기능 지원

USD는 복잡한 형식으로, 명세상 많은 기능을 지원하지만 모든 3D 소프트웨어가 이를 지원하는 것은 아닙니다. 예를 들어 OSX/iOS는 버텍스 색상을 지원하지 않습니다. USD 내보내기의 프리셋 모드는 OpenUSD, Apple, Procreate, ZBrush와의 호환성을 최선으로 추정하여 설정합니다.

::: 

## 렌더

프로젝트의 모든 설정(조명, 머티리얼, 후처리 등)을 합쳐 이미지를 내보냅니다. 

![](/images/file_render.webp)
### 미리보기

메뉴 제목 옆의 작은 미리보기 버튼을 누르면 툴바가 어두워져 최종 결과를 더 잘 미리볼 수 있습니다.

### 투명 배경
렌더에 알파 채널을 활성화합니다. 2D 프로그램에서 다른 이미지와 합성할 때 유용합니다. 부분 투명도는 지원되지 않는다는 점에 유의하세요.

### 인터페이스 표시
렌더에 Nomad의 UI를 포함하도록 활성화합니다.

### 렌더 비율
이미지 해상도에 곱해지는 배율입니다.

### 최종 크기
렌더에 사용할 해상도입니다. `Custom`을 선택하면 너비와 높이 슬라이더가 활성화됩니다. 

파일 메뉴가 활성화된 동안, 렌더 해상도가 화면 해상도와 일치하지 않으면 뷰포트에 점선 오버레이로 렌더 영역이 표시됩니다(정확한 표시를 위해서는 가로 모드여야 합니다).

### PNG 내보내기
이 버튼을 클릭하면 렌더 프로세스가 시작됩니다. 완료되면 이미지를 저장하거나 공유하는 방법을 선택할 수 있습니다.
