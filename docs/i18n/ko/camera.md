# ![](/icons/camera.webp) 카메라

이 메뉴에서는 카메라를 생성·수정하고, 카메라와 상호작용하는 방식을 제어할 수 있습니다.

![](/images/camera_overview2.webp)

Nomad에서 카메라는 다음과 같은 용도로 사용됩니다:

* 정밀한 각도에서 조각하기 위한 뷰 설정
* 피사체를 프레이밍하는 사진 카메라처럼 사용
* 장면을 탐색하기 위한 1인칭 시점 카메라
* 아이소메트릭 게임이나 산업용 렌더링 스타일을 위한 직교(Orthographic) 카메라

## 카메라 조작

### 회전
배경을 한 손가락으로 드래그하면 카메라가 회전합니다.  
모델 위에서 손가락을 드래그하면 조각 작업이 시작됩니다.

::: tip 배경을 터치할 수 없을 때도 카메라를 회전할 수 있나요?
네. 화면에 손가락 두 개를 올려(팬/줌 제스처를 시작하듯이) 둔 뒤, 손가락 하나를 떼면 됩니다.
:::

### 포커스 / 리셋
모델을 *더블 탭*하면 선택한 지점에 포커스를 맞춥니다.  
배경을 *더블 탭*하면 선택된 메시 전체에 카메라 포커스를 맞춥니다.

### 이동
손가락 *두 개*를 움직이면 카메라를 팬(이동)할 수 있습니다.

### 줌
핀치 제스처로 확대/축소할 수 있습니다.

### 롤(기울이기)
손가락 *두 개*를 회전시키면 뷰를 *롤*할 수 있습니다.
::: warning
이 제스처는 `trackball` 회전 모드에서만 사용할 수 있습니다.
:::

### 데스크톱 조작

데스크톱에서는 alt/opt 키로 카메라를 제어합니다:

* tip을 빈 공간에서 드래그 = 카메라 회전
* alt+tip 드래그 = 팬
* alt+tip 드래그 후 alt를 떼기 = 줌 (zbrush와 동일)

펜에 버튼이 2개 이상 있는 Wacom 태블릿의 경우, Wacom 설정에서 팁과 버튼을 다음과 같이 설정하세요:

* tip = 왼쪽 클릭
* 아래 로커 = 가운데 클릭
* 위 로커 = 오른쪽 클릭

이 설정을 사용하면 펜만으로 카메라를 조작할 수 있습니다:

* 위 로커 + 호버 이동 = 카메라 회전
* 아래 로커 + 호버 이동 = 팬

## 카메라 컨트롤

![](/images/camera_list.webp)

### 뷰
`Add View`를 사용해 카메라 뷰 포인트를 저장할 수 있습니다.  
뷰 이름을 클릭하면 해당 뷰로 카메라가 복원됩니다.

::: tip
저장된 뷰에는 [Projection Type](#projection-type) 설정과 [Reference image](background.md)도 함께 저장됩니다.  
서로 다른 배경을 가진 앞/왼쪽/뒤쪽 기준 뷰를 순환하고 싶을 때 유용합니다.
:::

| Action      | Icon                          | Description                                                                 |
| :---------: | :---------------------------: | :-------------------------------------------------------------------------: |
| Visibility  | ![](/icons/eye_open.webp)    | 카메라 표시 전환. 숨겨진 카메라는 이전/다음 버튼에서 건너뜁니다             |
| Name        |                               | 카메라 선택                                                                 |
| Image       | ![](/icons/image.webp)       | 카메라에 참조 이미지가 연결되어 있을 경우 썸네일 표시                       |

| Delete      | ![](/icons/trash.webp)       | 카메라 삭제                                                                 |

### ![](/icons/tool_view.webp) Add View
현재 뷰를 기반으로 새 카메라를 생성합니다.

### ![](/icons/camera.webp) Icons

뷰포트에 카메라 아이콘을 표시할지 전환합니다. 카메라가 선택된 경우 해당 아이콘은 항상 표시됩니다.

### Projection Type
카메라의 `Field of View`(FOV / 초점 거리)를 변경할 수 있습니다.  
비율을 파악하기 위해 조각 작업에서는 낮은 FOV를 사용하는 것이 일반적으로 권장됩니다.  
또한 `Orthographic` 모드를 사용할 수 있으며, 이는 FOV가 0인 것과 거의 비슷하게 동작합니다.

### First Person
피벗을 조각이 아닌 카메라 위치에 직접 두도록 설정합니다.  
배경을 한 손가락으로 드래그하면 카메라 위치는 고정된 채 회전만 변경되며, 1인칭 게임과 비슷하게 동작합니다.  
단일 오브젝트가 아닌 환경을 조각할 때 유용합니다.

![](/images/camera_rotation_ortho_view.webp)

### Rotation Type
기본적으로 카메라는 `Turntable` 회전 모드를 사용합니다.  
이 모드는 자유도가 두 축으로 제한되어 더 직관적이지만, 상황에 따라 더 높은 유연성이 필요할 수 있습니다.  
`Trackball` 모드로 전환하면, 뷰포트에서 손가락 *두 개*를 회전시켜 뷰를 *롤*할 수 있습니다.  
데스크톱에서는 일부 사용자에게 더 익숙할 수 있는 대체 트랙볼 모드도 있습니다.

### Orthographic snap

활성화하면, 키보드에서 Shift를 누른 채로 뷰를 회전할 때 카메라가 가장 가까운 앞/뒤/위/아래/왼쪽/오른쪽 뷰로 스냅되며, 카메라가 직교(Orthographic) 모드로 전환됩니다.  
또한 뷰 큐브를 클릭해 앞/뒤/왼쪽/오른쪽/위/아래로 스냅할 때도 카메라는 직교 모드가 됩니다.

### Reset view

카메라를 정면으로 이동시키고, 장면 전체가 화면에 맞도록 조정합니다.

### Snap view
가장 가까운 앞/뒤/왼쪽/오른쪽/위/아래 뷰로 스냅합니다.  
이미 이들 중 하나의 뷰에 있는 경우, 다시 클릭하면 180도 반대 방향으로 스냅됩니다.

### Speed

카메라가 너무 느리거나 빠르게 움직인다고 느껴진다면, `rotation`, `translation`, `zooming`에 대한 속도 배율을 조정할 수 있습니다.  
조각의 크기가 매우 크거나 작을 때 유용합니다.

### 피벗 개요

카메라를 회전할 때 작은 분홍색 점이 보이는데, 이것이 카메라 피벗 포인트입니다.  
피벗이 어디에 있는지 이해하는 것은 카메라 때문에 길을 잃거나 답답함을 느끼지 않기 위해 매우 중요합니다.

기본적으로 피벗은 다음 동작을 통해 업데이트됩니다:
- 모델을 더블 탭
- 배경을 더블 탭 (새 피벗은 메시의 중심에 위치)
- 화면에 손가락 *두 개*를 올려(팬/줌/롤) 사용하면, *두* 손가락의 중심에 피벗이 업데이트됨

### Update Pivot...

다음 옵션을 통해 피벗이 업데이트되는 방식을 추가로 커스터마이즈할 수 있습니다:

* 카메라가 움직이기 시작할 때
* 공중 피벗 허용
* 더블 탭 시
* 조각(스트로크)

::: tip
사용법에 익숙해지면 [Settings](settings.md) 메뉴에서 힌트용 분홍색 점을 숨길 수 있습니다.
:::

### Double tap on object
`Focus`가 활성화된 경우, 오브젝트를 더블 탭하면 피벗이 탭한 오브젝트로 이동합니다.

### Double tap on background
활성화된 경우, 피벗을 Selection, Scene 중 하나로 설정하거나 이 둘 사이를 토글합니다.
