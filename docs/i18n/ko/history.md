# ![](/icons/history.webp) 히스토리
![](/images/history_overview.webp)

대부분의 콘텐츠 제작 도구와 마찬가지로 Nomad에서도 모든 편집 작업을 실행 취소/다시 실행할 수 있습니다.  
되돌릴 수 있는 작업 수에는 제한이 있지만, 이 동작은 사용자가 제어할 수 있습니다.

::: tip
빠른 제스처로 실행 취소/다시 실행을 할 수 있습니다:
- 두 손가락 탭: 실행 취소
- 세 손가락 탭: 다시 실행
:::

## 히스토리
![](/images/history_history.webp)

이 패널은 히스토리 스택을 표시하며, 단계 수, 작업 이름, 그리고 해당 단계가 사용하는 메모리 양을 보여줍니다.

## 설정
![](/images/history_settings.webp)

### 히스토리 제한 (Mb)
히스토리 스택이 이 값을 초과하면, 메모리 사용량이 이 한도에 맞도록 오래된 작업들이 제거됩니다.

### 최대 실행 취소 가능 횟수
되돌릴 수 있는 최대 작업 수를 제어할 수 있습니다.

## 카메라 복원
각 작업마다 카메라의 시점이 저장됩니다.  
이 옵션을 활성화하면, 작업을 실행 취소하거나 다시 실행할 때 카메라가 저장된 시점으로 되돌아갑니다.

## 포함할 액션

* `Lights` - 비활성화하면, 기즈모 이동을 제외한 조명 관련 작업은 히스토리 스택에서 무시됩니다
* `Matcaps & HDRIs` - 비활성화하면, matcap 및 hdri 변경 사항은 히스토리 스택에서 무시됩니다
* `PostProcess` - 비활성화하면, 후처리 옵션 변경 사항은 히스토리 스택에서 무시됩니다

## 메모리 통계

이 섹션은 Nomad에서 사용 중인 메모리의 상세 내역을 제공합니다.
