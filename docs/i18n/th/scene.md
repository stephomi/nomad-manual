# ![](/icons/scene.webp) ซีน (Scene) 

เมนูนี้ใช้จัดการอ็อบเจ็กต์ แสง กล้อง และรีพีตเตอร์ใน Nomad โดยจะแสดงลำดับชั้นของซีนในรูปแบบมุมมองต้นไม้ (tree-view) ทำให้คุณสามารถแก้ไขหลายๆ ด้านของอ็อบเจ็กต์ได้ นอกจากนี้ยังใช้สร้างอ็อบเจ็กต์ใหม่ รวมถึงรวม (join) และแยก (separate) อ็อบเจ็กต์ได้หลายรูปแบบ

### การจัดการ Tree view

กดค้างบนรายการใน tree view จนกว่ารายการนั้นจะเปลี่ยนเป็นสีเหลือง จากนั้นคุณสามารถเลื่อนขึ้นหรือลงใน tree view ได้ รวมถึงลากไปวางบนรายการอื่นเพื่อทำให้มันกลายเป็นลูก (child) ของรายการนั้น

เมื่อเลือกหลายรายการ ส่วนใหญ่จะเป็นสีเหลืองเข้ม และจะมีหนึ่งรายการเป็นสีเหลืองอ่อน กดค้างและลากที่รายการสีเหลืองอ่อนเพื่อย้ายอ็อบเจ็กต์ทั้งหมดพร้อมกัน

เมื่อคุณเลือกอ็อบเจ็กต์แม่ (parent) โดยค่าเริ่มต้นอ็อบเจ็กต์ลูกทั้งหมดจะถูกเลือกด้วย การแตะที่ไอคอนของ parent จะสลับระหว่างการเลือกเฉพาะ parent หรือเลือกทั้ง parent และ children

### การเลือกหลายอ็อบเจ็กต์ (Multiselection)
คุณสามารถเลือกหลายอ็อบเจ็กต์เพื่อช่วยทำสองอย่าง:
- ใช้เครื่องมือ gizmo เพื่อย้ายหลายอ็อบเจ็กต์พร้อมกัน
- รวมอ็อบเจ็กต์ด้วยการ Join และ Boolean

ทำได้โดยใช้ช่องทำเครื่องหมาย `Multiselection` แล้วคลิกที่อ็อบเจ็กต์ในลิสต์

::: tip Quick multiselection
คุณยังสามารถเลือกหลายอ็อบเจ็กต์ใน viewport ได้โดยกดปุ่มลัด `Smooth` ค้างไว้แล้วแตะบน mesh อื่น

คุณสามารถยกเลิกการเลือกอ็อบเจ็กต์ได้โดยแตะที่มันอีกครั้ง (เฉพาะเมื่อมีการเลือกมากกว่าหนึ่งอ็อบเจ็กต์)
:::

::: warning ข้อจำกัดของ gizmo
เมื่อใช้ multiselection เครื่องมือ gizmo จะไม่สนใจ masking เสมอ
และการสเกลแบบ X/Y/Z จะถูกปิดใช้งาน

เหตุผลคือ multiselection อนุญาตเฉพาะการแปลงทั้ง mesh ไม่ใช่การแปลงระดับจุดยอด (per-vertex)
อาจมีการปรับปรุงในอนาคต
:::


## Join
ตัวเลือกนี้จะรวมอ็อบเจ็กต์ที่เลือกหลายตัวให้กลายเป็นรายการอ็อบเจ็กต์เดียว

คุณสามารถดูตัวอย่างวิดีโอได้ในหัวข้อ [Separate](#separate)

## Boolean
![](/images/scene_boolean_menu.webp) 
รวมอ็อบเจ็กต์ให้กลายเป็นพื้นผิวเดียว

`Voxel merge` จะคงปริมาตรของอ็อบเจ็กต์ไว้ และคำนวณโพลิกอนใหม่ที่มีระยะห่างสม่ำเสมอบนพื้นผิว เนื่องจากมีขั้นตอนการคำนวณ Voxel merge จึงรองรับเรขาคณิตที่ซับซ้อนได้ดี แต่ก็อาจสูญเสียรายละเอียดเล็กๆ ถ้าความละเอียดเป้าหมายไม่สูงพอ

`Boolean` จะพยายามคงโครงสร้างโพลิกอนเดิมไว้ และเย็บโพลิกอนตรงบริเวณที่อ็อบเจ็กต์ทับซ้อนกัน วิธีนี้ให้ผลลัพธ์ที่สะอาดและคมกว่าการใช้ voxel merge อย่างไรก็ตามจำเป็นต้องใช้ mesh แบบ “ปิดสนิท” (watertight) คือห้ามมีรูหรือรูปทรงผิดปกติ ถ้าทำไม่สำเร็จ มักจะใช้ voxel merge แทนได้

### การทำงานของ Boolean
ทั้ง Voxel Merge และ Boolean จะใช้สถานะการมองเห็นของอ็อบเจ็กต์เพื่อควบคุมประเภทของการทำงาน:

#### Union
เมื่ออ็อบเจ็กต์ทั้งสองมองเห็นได้ จะสร้าง boolean **union** คือรวมผิวด้านนอกของอ็อบเจ็กต์เข้าด้วยกัน โดยไม่มีพื้นผิวด้านใน ![](/images/boolean_union.webp)

#### Subtract
เมื่ออ็อบเจ็กต์หนึ่งถูกซ่อน = boolean **subtract** อ็อบเจ็กต์ที่มองไม่เห็นจะถูกหักออกจากอ็อบเจ็กต์ที่มองเห็น ![](/images/boolean_subtract.webp)

#### Intersect
เมื่ออ็อบเจ็กต์ทั้งสองถูกซ่อน = boolean **intersection** สร้างรูปทรงใหม่เฉพาะบริเวณที่อ็อบเจ็กต์ทั้งสองทับซ้อนกัน ![](/images/boolean_intersect.webp)


### ปุ่ม Voxel Merge
การกดปุ่มนี้จะทำการ voxel merge กับอ็อบเจ็กต์ที่เลือก ถ้าทำกับอ็อบเจ็กต์เดียวจะเป็นการ retopologize ให้เป็นโพลิกอนที่มีระยะห่างสม่ำเสมอ เหมาะเมื่ออ็อบเจ็กต์มีโพลิกอนที่ถูกยืด

### Resolution
ความละเอียดของกริด voxel 3 มิติที่ใช้ในการคำนวณ เมื่อเปลี่ยนค่าจะมีลายตารางหมากรุกแสดงบนอ็อบเจ็กต์เพื่อพรีวิวขนาดโพลิกอน

### Build multiresolution
สร้างระดับ multiresolution ที่ต่ำกว่าความละเอียดเป้าหมาย เช่น ถ้าความละเอียดคือ 400 และ build multiresolution เป็น 3 คุณจะได้ mesh ใหม่ที่มีประมาณ 296,000 โพลิกอน และมีระดับ subdiv ต่ำกว่านั้นอีก 3 ระดับที่ 74,000, 18,000, 4,000k

### Keep sharp edges
เปิดการ snap mesh voxel เข้าหาขอบ ทำงานได้ดีที่สุดกับรูปทรงง่ายๆ

### ปุ่ม Boolean
การกดปุ่มนี้จะทำ polygon boolean โดยใช้ไลบรารี Manifold โดย Emmett Lalish

## Add menu

![](/images/scene_addmenu_overview.webp)

เมนูนี้ใช้สร้าง primitive, group, กล้อง, รีพีตเตอร์ และแสง

Primitive คือรูปทรงพื้นฐานที่สามารถปรับได้ด้วยพารามิเตอร์ เมื่อคุณปรับ primitive ตามต้องการแล้ว ให้ “validate” เพื่อแปลงพารามิเตอร์เหล่านั้นให้กลายเป็น polygon mesh ที่สามารถปั้น (sculpt) และเพนต์ได้ Primitive จะไม่สามารถปรับพารามิเตอร์ได้อีกหลังจาก validate แล้ว


![](/images/scene_addmenu_top.webp)

### On gizmo
เปิดเพื่อวาง primitive ใหม่ไว้ที่ตำแหน่งของรูปทรงหรือ gizmo ที่เลือกอยู่ปัจจุบัน ถ้าปิด primitive จะถูกวางที่จุดกึ่งกลางของซีน

### Select gizmo
เปิดเพื่อสลับไปใช้เครื่องมือ gizmo โดยอัตโนมัติเมื่อสร้าง primitive ใหม่ 

### Advanced

เมนูนี้ให้คุณตั้งค่าความชอบว่าต้องการให้ primitive, group, repeater ใหม่ถูกสร้างที่ไหน อาจเป็นบนอ็อบเจ็กต์ที่เลือก ที่จุดกำเนิดของโลก (world origin) หรือที่ตำแหน่งของ gizmo


### UV's
เปิด UV บน primitive UV (มักเรียกว่า texture coordinates) คือข้อมูลเพิ่มเติมใน 3D ที่ใช้สำหรับวางเท็กซ์เจอร์บนพื้นผิว ใช้หน่วยความจำเพิ่มขึ้น แต่สำหรับอุปกรณ์ส่วนใหญ่จะไม่เป็นปัญหายกเว้นเมื่อจำนวนโพลิกอนสูงมาก (เช่น 10 ล้านโพลิกอนขึ้นไป)

แต่ base mesh ใน Nomad ไม่ได้ใช้ `Project on sphere` หมายความว่ามันไม่กลมสมบูรณ์
:::

### แถบเครื่องมือ Primitive

![](/images/scene_primitive_toolbar.gif)

เมื่อสร้าง primitive แล้ว แถบเครื่องมือจะปรากฏขึ้นเพื่อควบคุมพารามิเตอร์ของมัน

* `Validate` แปลง primitive ให้เป็นอ็อบเจ็กต์มาตรฐานเพื่อให้สามารถปั้นและเพนต์ได้
* `Edit` สลับการแสดง gizmo ของ primitive ซึ่งจะแสดงบน primitive โดยตรงเพื่อควบคุมพารามิเตอร์ เช่น ความกว้างของกล่อง หรือรัศมีรูของ cylinder
* `Mirror` สลับการวาง mirror repeater ไว้เหนือ primitive
* `...` แสดงเมนูของ primitive

Primitive แต่ละชนิดจะมีตัวเลือกเพิ่มเติมบนแถบเครื่องมือ:

* `Project` ทรงกลมถูกสร้างจากกล่องที่ถูก subdivide ซึ่งเหมาะกับการปั้น แต่ทำให้มันไม่กลมสมบูรณ์ ตัวเลือกนี้จะบังคับให้รูปทรงเข้าใกล้ทรงกลมที่สมบูรณ์มากขึ้น Icosahedron ก็ใช้ตัวเลือกนี้เช่นกัน
* `Cap` สลับการปิดปลายของรูปทรง เช่น cylinder สามารถมีฝาปิดด้านบน ด้านล่าง ทั้งสองด้าน หรือไม่มีเลย
* `Hole` สลับการสร้างรูทะลุกลางรูปทรง โดยจะวนลูประหว่าง ไม่มีรู, รูที่มีรัศมีเดียว, หรือรูที่มีรัศมีต่างกันด้านบนและด้านล่าง
* `Radius` สลับว่า cylinder จะใช้รัศมีเดียว หรือใช้รัศมีต่างกันที่ด้านบนและด้านล่าง
* `Disk` สลับว่า plane จะเป็นรูปสี่เหลี่ยมหรือรูปวงกลม

แถบเครื่องมือเล็กด้านล่างแถบหลักใช้สลับระหว่าง gizmo ของ primitive และ gizmo สำหรับการแปลง (transform)

::: tip

การคลิกที่ชื่อของแถบเครื่องมือจะสลับตำแหน่งระหว่างด้านบนและด้านล่างของหน้าจอ การคลิกที่ลูกศรตรงมุมจะยุบแถบเครื่องมือ

:::


### เมนู Primitive

![](/images/scene_primitive_menu.webp)

เมนูนี้มีพารามิเตอร์ทั้งหมดของ primitive ที่เลือก พารามิเตอร์คือคำอธิบายพื้นฐานของรูปทรง เช่น ในการอธิบายแหวน คุณจะอธิบายรัศมีด้านนอก รัศมีด้านใน และจำนวนโพลิกอน

พารามิเตอร์ส่วนใหญ่ควรเข้าใจได้ไม่ยาก และมีพารามิเตอร์ร่วมที่ใช้กับ primitive ทุกชนิด:

* `UVs` ปุ่ม UVs เล็กๆ ด้านบนเมนูใช้สลับการสร้างพิกัด UV
* `Validate` ปุ่ม validate เล็กๆ ใช้แปลง primitive ให้เป็นอ็อบเจ็กต์มาตรฐานเพื่อให้สามารถปั้นและเพนต์ได้
* `Max faces` ตั้งขีดจำกัดจำนวนโพลิกอนสูงสุดของอ็อบเจ็กต์เพื่อหลีกเลี่ยงการทำให้อุปกรณ์ค้าง
* `Post subdivision` เปิดจำนวน subdivision ที่เลือกจากส่วน multiresolution ในเมนู topology ใช้สร้าง primitive ที่มีมุมโค้งมนและนุ่ม โดยใช้ division topology ต่ำ ตัวอย่างเช่น ตั้ง division ของ box เป็น 2 และ post subdivision เป็น 4 จะได้กล่องที่มีมุมโค้งมน
* `Linear subdivision` ตั้งจำนวนระดับของ linear subdivision ก่อนใช้ smooth subdivision ปกติ ใช้ควบคุมว่ามุมของพื้นผิวที่ subdivide จะคมแค่ไหน เช่น ตั้ง division ของ box เป็น 2, post subdivision เป็น 4 แล้วลองเปลี่ยน linear subdivision ระหว่าง 0 ถึง 4 มุมของกล่องจะเปลี่ยนจากนุ่มเป็นคม

### Topology

ควบคุมจำนวนโพลิกอนใน primitive โดยปกติสไลเดอร์จะเชื่อมกัน ดังนั้นการเปลี่ยนสไลเดอร์ที่ใช้งานอยู่จะปรับจำนวนโพลิกอนทั้งหมดอย่างสมดุล คุณสามารถแตะปุ่ม unlink เพื่อควบคุม division ในแกน X/Y/Z แยกจากกัน

### Geometry

ควบคุมขนาดโดยรวมของ primitive ในหน่วย X/Y/Z สำหรับรูปทรงเหลี่ยม และในรัศมีสำหรับรูปทรงกลม


### UV Sphere
::: warning
UV Sphere ไม่เหมาะกับการปั้น โดยเฉพาะบริเวณขั้ว (poles)

ควรใช้ primitive [Sphere](#sphere), [Box](#box) หรือ [Icosahedron](#icosahedron) ร่วมกับตัวเลือก `Project on sphere` แทน

อย่างไรก็ตาม topology อาจพอใช้ปั้นได้ถ้าคุณตั้งค่า `Division` ให้ต่ำมาก  
จากนั้นใช้สไลเดอร์ `Overall Subdivision` เพื่อเพิ่มจำนวนโพลิกอน

แม้จะไม่เหมาะกับการปั้นทั่วไป แต่ UV Sphere มีประโยชน์สำหรับทำดวงตา ถ้าคุณหมุนให้ขั้วของทรงกลมอยู่ที่ตำแหน่งรูม่านตา (pupil) โครงสร้างโพลิกอนจะเหมาะกับการเพนต์และปั้นม่านตาและรูม่านตาโดยธรรมชาติ
:::


### Triplanar
Primitive นี้มีความพิเศษตรงที่คุณควรใช้ [Masking tool](tools.md#mask) กับมันเพื่อขึ้นรูปเรขาคณิต

![](/videos/triplanar.mp4)


::: tip
แตะสองครั้งบนระนาบใดระนาบหนึ่ง กล้องจะโฟกัสไปที่ระนาบนั้น  
อย่างไรก็ตามจะใช้ไม่ได้ถ้าคุณหมุน primitive ด้วย gizmo
:::

Triplanar ใช้ข้อมูล mask จาก 3 ระนาบเพื่อเติม voxel grid แล้วจึงแปลงเป็นโพลิกอน (polygonize) (โดยใช้ [Voxel Remesher](topology.md#voxel-remeshere))

แต่ละระนาบมีระนาบสมมาตรของตัวเอง

::: warning
ทุกครั้งที่คุณอัปเดตขนาดของ Triplanar คุณภาพของการเพนต์ mask จะลดลง

ตอนนี้ยังไม่มีตัวเลือกในการ “ล็อก” การเพนต์บนระนาบเดียว แต่อาจมีในอนาคต  
คุณสามารถใช้ [Connected Topology](stroke.md#connected-topology) เพื่อช่วยเล็กน้อย กล่าวคือถ้าเคอร์เซอร์ของคุณอยู่บนระนาบใดระนาบหนึ่งอย่างแม่นยำ มันจะไม่กระทบระนาบอื่น
:::

## Repeaters
![](/images/scene_primitive_repeaters.webp)

Repeater คือโหนดที่สร้าง instance ของอ็อบเจ็กต์ที่อยู่ใต้มัน 

### Array
![](/images/scene_primitive_array.webp)

เมื่อมีอ็อบเจ็กต์เป็นลูกของโหนดนี้ จะสามารถสร้าง instance เป็นรูปแบบกริดได้ เมื่อเลือกแล้วจะมีตัวควบคุมดังนี้:
* Fit inside - สลับระหว่างการควบคุมขนาดของกริด/กล่องของ array หรือควบคุมระยะห่างระหว่าง instance
* CountX/Y/Z - จำนวน instance ในแต่ละแกน
* OffsetX/Y/Z - ระยะห่างระหว่าง instance เมื่อปิด fit inside
* SizeX/Y/Z - ความกว้าง/สูง/ลึกของกริด array ทั้งหมดเมื่อเปิด fit inside

### Curve
![](/images/scene_primitive_curve.webp)
สร้างเส้นโค้ง โดยอ็อบเจ็กต์ลูกของโหนดนี้จะถูกทำซ้ำตามเส้นโค้ง เมื่อเลือกแล้วจะมีตัวควบคุมดังนี้:
* Edit - อนุญาตให้เพิ่มจุดบนเส้นโค้ง และย้ายจุดบนเส้นโค้ง
* Snap - snap จุดของเส้นโค้งเข้ากับเรขาคณิตอื่น
* Align - หมุนรูปทรงลูกให้จัดแนวไปตามทิศทางของเส้นโค้ง
* Count - จำนวน instance
* Closed - สลับให้เส้นโค้งปิดปลายเชื่อมกัน หรือเป็นเส้นเปิด
* Radius - เปิดตัวควบคุมที่แต่ละจุดของเส้นโค้งเพื่อควบคุมสเกลของ instance
* Twist - เปิดตัวควบคุมที่แต่ละจุดของเส้นโค้งเพื่อควบคุมการหมุนบิดของ instance 
* B-spline - สลับให้ instance ตามเส้นโค้งแบบตรงๆ หรือใช้การอินเตอร์โพลแบบ b-spline ซึ่งให้ผลลัพธ์ที่นุ่มนวลกว่า 

### Radial
![](/images/scene_primitive_radial.webp)

อ็อบเจ็กต์ลูกของโหนดนี้จะถูกทำซ้ำเป็นวงกลม ย้ายอ็อบเจ็กต์ลูกเพื่อเปลี่ยนรัศมีของ repeater นี้ เมื่อเลือกแล้วจะมีตัวควบคุมดังนี้:
* RadialX/Y/Z - การเลือกปุ่มเหล่านี้จะตั้งแกนของ radial และตั้งจำนวน instance



### Mirror
![](/images/scene_primitive_mirror.webp)

อ็อบเจ็กต์ลูกของโหนดนี้จะถูกสะท้อนข้ามแกน เมื่อเลือกแล้วจะมีตัวควบคุมดังนี้:
* Gizmo - เปิด gizmo สำหรับการแปลงเพื่อกำหนดจุดศูนย์กลางของ mirror สามารถหมุนและสเกลได้ เมื่อเสร็จแล้วให้แตะ gizmo อีกครั้งเพื่อกลับไปยังตัวควบคุมปกติ
* X/Y/Z - ตั้งระนาบ mirror

Repeater ทุกตัวมีตัวควบคุม `Validate` ซึ่งใช้แปลงผลลัพธ์ของ repeater และจะถามว่าจะ bake อย่างไร:
* Join children - รวม instance ทั้งหมดเป็นอ็อบเจ็กต์เดียว
* Keep instances - คง instance ไว้เป็น instance แต่ไม่อยู่ใต้ repeater อีกต่อไป
* Un-instances - แปลง instance ให้เป็นอ็อบเจ็กต์ที่ไม่ซ้ำกัน

::: tip Tip: ผสมผสาน Repeaters
Repeater สามารถเป็นลูกของ repeater อื่นได้ และหลายอ็อบเจ็กต์สามารถเป็นลูกของ repeater ได้เช่นกัน ทำให้เกิดเอฟเฟกต์ที่ซับซ้อนได้

![](/images/scene_repeater_combine.webp)
:::

::: tip Tip: จุดหมุนของ Repeater

Repeater บางชนิดจะพยายามตั้งจุดหมุนของอ็อบเจ็กต์ลูกโดยอัตโนมัติ ดังนั้นแม้คุณจะย้ายหรือหมุนด้วย gizmo อ็อบเจ็กต์ก็อาจไม่ขยับ ถ้าคุณต้องการ override พฤติกรรมนี้ ให้แทรก group ระหว่าง repeater กับอ็อบเจ็กต์ลูก จากนั้นคุณจะสามารถย้ายรูปทรงลูกได้อย่างอิสระจาก repeater
:::
