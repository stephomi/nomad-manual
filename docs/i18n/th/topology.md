# ![](/icons/multires.webp) ทอปอโลยี {#topology}

เมนูนี้ควบคุมทอพอโลยีของอ็อบเจ็กต์ใน Nomad รวมถึงเครื่องมือสำหรับเบกและถ่ายโอนรายละเอียดระหว่างอ็อบเจ็กต์ และระหว่างเท็กซ์เจอร์

![](/images/topology_overview.webp)

Nomad ทำงานบนพื้นฐานของโพลิกอน โดยใช้สามเหลี่ยมและควอดในการจัดการเรขาคณิต  
เมื่อพูดถึงทอพอโลยี เราหมายถึงทั้งจำนวนหน้า (faces) และวิธีที่จุด (vertices) เชื่อมต่อกัน

การติดตามทอพอโลยีเป็นเรื่องสำคัญ โดยเฉพาะหากคุณต้องการปั้นหรือเพนต์รายละเอียดที่ละเอียดมาก

::: tip จะติดตามทอพอโลยีของคุณอย่างไร?
คุณสามารถแสดง [Wireframe](settings.md#wireframe) หรือเพียงแค่ปิด [Smooth Shading](settings.md#smooth-shading)
:::

## สถิติโพลิกอน {#polygon-stats}

![](/images/topology_stats.webp)

ส่วนบนของเมนูทอพอโลยีจะแสดงข้อมูลโพลิกอนของอ็อบเจ็กต์ที่เลือกและทั้งซีน ตัวเลขจะแสดงจำนวน vertices ทั้งหมด จำนวนสามเหลี่ยมทั้งหมด และจำนวนควอด (โพลิกอน 4 ด้าน) ทั้งหมด

แตะที่ส่วนนี้เพื่อแสดงรายการสถิติโพลิกอนของอ็อบเจ็กต์โพลิกอนทั้งหมดในซีน

## ![](/icons/multires.webp) มัลติเรโซลูชัน {#multiresolution}

![](/images/topology_multires_menu.webp)

### มัลติเรโซลูชันคืออะไร? {#what-is-multiresolution}
ฟีเจอร์ multiresolution มีประโยชน์หลักในสองกรณี:
- ใช้อัลกอริทึม smooth subdivision เพื่อเพิ่มจำนวนโพลิกอนของอ็อบเจ็กต์
- จัดการหลายระดับความละเอียดเพื่อให้คุณสลับระหว่างการแก้ไขระดับใหญ่กับระดับเล็กได้

![](/videos/multiresolution.mp4)

#### เวิร์กโฟลว์มัลติเรโซลูชัน {#multiresolution-workflow}
จุดสำคัญอย่างหนึ่งของ multiresolution คือคุณสามารถย้อนกลับไปที่ความละเอียดต่ำกว่า แก้ไขอ็อบเจ็กต์ แล้วกลับไปที่ความละเอียดสูงสุดได้โดยไม่สูญเสียรายละเอียดความละเอียดสูง รายละเอียดทั้งหมดจะถูก project กลับให้อัตโนมัติ

::: warning
หากคุณใช้เครื่องมือที่เปลี่ยนทอพอโลยีของอ็อบเจ็กต์ คุณจะสูญเสียทุกระดับความละเอียดอื่นของอ็อบเจ็กต์นั้น!  
คุณควรได้รับคำเตือนทุกครั้งที่อาจเกิดเหตุการณ์นี้ เช่น เมื่อคุณใช้:
- [Voxel Remesher](#voxel-remesher)
- [Dynamic Topology](#dynamic-topology)
- [Trim tool](tools.md#trim)
- [Split tool](tools.md#split)
:::


### สไลเดอร์มัลติเรโซลูชัน {#multiresolution-slider}
สไลเดอร์นี้แสดงจำนวนระดับ subdivision ของอ็อบเจ็กต์ปัจจุบัน หากมีแท่งแนวตั้ง 6 แท่ง แสดงว่ามี 6 ระดับ subdivision วงกลมจะแสดงระดับ subdivision ที่กำลังแสดงอยู่

### ย้อนกลับ {#reverse}
เมื่ออยู่ที่ระดับ subdivision ต่ำสุด ปุ่ม Reverse จะพยายามสร้างระดับที่ต่ำกว่าปัจจุบัน โดยทั่วไปจะทำได้ก็ต่อเมื่ออ็อบเจ็กต์ถูกสร้างด้วย subdivision ตั้งแต่แรก เช่น ใน Nomad หรือแอป 3D อื่นที่รองรับ multiresolution subdivision surfaces

### แบ่งส่วนย่อย {#subdivide}
ปุ่ม *Subdivide* จะเพิ่มจำนวนโพลิกอนเป็น 4 เท่า ดังนั้นควรจับตาดู polycount ไว้เพราะมันเพิ่มขึ้นเร็วมาก!  
จุดสำคัญของ *Subdivision Surface* คือมันจะลู่เข้าไปสู่ *Smooth Surface*  
เพื่อทำความเข้าใจ ลองใช้ปุ่ม *Subdivide* กับอ็อบเจ็กต์ที่มีโพลิกอนเพียงไม่กี่หน้า

คุณสามารถปิดพฤติกรรม *Smooth* นี้ได้โดยติ๊กตัวเลือก `Linear subdivision`

### ลบเลเวลต่ำ {#delete-lower}
หากมี subdivision ที่ต่ำกว่าระดับที่กำลังแสดงอยู่ ให้ลบระดับเหล่านั้น หากลบโดยไม่ตั้งใจ คุณสามารถสร้างใหม่ได้ด้วยปุ่ม Reverse

### ลบเลเวลสูง {#delete-higher}
หากมี subdivision ที่สูงกว่าระดับที่กำลังแสดงอยู่ ให้ลบระดับเหล่านั้น

### การแบ่งส่วนเชิงเส้น {#linear-subdivision}
Subdivide mesh โดยไม่ใช้การ smoothing

### ขอบคม {#sharp-border}
หากอ็อบเจ็กต์ของคุณมี facegroup เมื่อเปิดตัวเลือกนี้จะทำให้ขอบของ facegroup คมอยู่ สามารถตั้งค่านี้แยกในแต่ละระดับ subdivision ได้ (สไลเดอร์ subdivision จะมีไอคอนเล็ก ๆ เหนือระดับนั้นเพื่อบอกสถานะ)

### คงรูปสามเหลี่ยม {#keep-triangles}
ระบบ subdivision surface มาตรฐานส่วนใหญ่จะพยายามแปลงโพลิกอนทั้งหมดให้เป็นควอดระหว่างการ subdivide ตัวเลือกนี้จะบังคับให้ subdivision ใช้สามเหลี่ยมแทน

### ล็อก (LV0) {#lock-lv0}

ป้องกันไม่ให้ระดับ subdivision ต่ำสุดถูกแก้ไข ซึ่งสำคัญหากอ็อบเจ็กต์ของคุณถูกสร้างจากแอปอื่น และต้องการให้ base mesh ไม่เปลี่ยนแปลง เมื่อปิดตัวเลือกนี้ การเปลี่ยนแปลงขนาดใหญ่ที่ระดับ subdivision สูงกว่าจะทำให้ระดับ 0 เคลื่อนตาม

::: tip 

โดยค่าเริ่มต้น Subdivision จะทำให้ขอบคมทั้งหมดนุ่มลง หากต้องการให้ขอบยังคมเล็กน้อย ลองใช้ linear subdivision หรือ Sharp border ใน 2 ระดับ subdivide แรก แล้วค่อยปิดในระดับที่สูงกว่า วิธีนี้จะสร้าง mesh ที่คมกึ่ง ๆ หลัง subdivide

:::


## ![](/icons/voxel.webp) ว็อกเซลรีมีชเชอร์ {#voxel-remesher}
![](/images/topology_voxel_menu.webp)
เมื่อใช้ `Voxel Remesher` ทั้ง mesh จะถูกบังคับให้มีทอพอโลยีที่มีความละเอียดสม่ำเสมอ หมายความว่าโพลิกอนทั้งหมดจะมีขนาดใกล้เคียงกันมาก วิธีนี้มีประโยชน์มากเมื่อคุณไม่อยากคิดเรื่องทอพอโลยีและแค่อยากปั้นแบบ free-form

เวิร์กโฟลว์การปั้นทั่วไปอาจเริ่มจากการใช้ `Voxel Remesher` เพื่อบล็อกทรงหยาบ ๆ ด้วยความละเอียดต่ำ  
เพียงกดปุ่ม *Remesh* เป็นระยะเมื่อคุณดึง mesh จนยืดมากเกินไปเพื่อหลีกเลี่ยงการบิดเบือนมากเกินไป

![](/videos/voxel_remesher.mp4)


::: tip Voxel?
ดังที่กล่าวไว้ข้างต้น Nomad เป็นซอฟต์แวร์แบบโพลิกอน แต่ `Voxel Remesher` เป็นข้อยกเว้นที่ใช้แนวทางอื่น (ชั่วคราว) เพื่อทำ remesh

ความแตกต่างสำคัญคือวิธี voxel จะไม่ยอมรับการ self-intersection ดังนั้นส่วนที่ตัดกันเองจะถูกแก้ไข  
และยังไม่รองรับ mesh ที่มีรู

คำว่า “รู” ที่นี่ไม่ได้หมายถึง `genus hole` (รูของโดนัท) แต่หมายถึง mesh ที่ไม่ `watertight`/`closed`  
โดยทั่วไปหมายความว่าก่อน remesh รูทุกจุดจะถูกปิด คล้ายกับ [Trim tool](tools.md#trim) หรือ [ฟีเจอร์เติมรู](scene.md#hole-filling)
:::

### รีมีช {#voxel-remesh}
รัน voxel remesh

### ความละเอียด {#voxel-resolution}
ขนาดของ voxel ที่ใช้ระหว่างการคำนวณ ขณะเปลี่ยนพารามิเตอร์นี้จะมีลายตารางหมากรุกซ้อนบน mesh เพื่อแสดงตัวอย่างผลลัพธ์

### สร้างมัลติเรโซลูชัน {#build-multiresolution}
สร้างระดับ multiresolution ที่ต่ำกว่าสำหรับ voxel remesh หากคุณใช้ลายตารางเพื่อกำหนด resolution และตั้ง build multiresolution เป็น 2 ผลลัพธ์สุดท้ายจะมีรายละเอียดตรงกับสไลเดอร์ resolution และหากไปที่แท็บ multires จะอยู่ที่ระดับ 2 หมายความว่าคุณจะมี multires mesh ความละเอียดต่ำกว่าที่ระดับ 1 และ 0 วิธีนี้เป็นวิธีที่ดีในการสร้าง mesh ที่สะอาดและโพลิกอนสม่ำเสมอ พร้อมทั้งมี control mesh ความละเอียดต่ำกว่า

::: tip Tip: Build multiresolution และ stable smoothing

ตัวเลือกนี้บางครั้งอาจทำให้เกิด ‘loop’ ในเรขาคณิตที่ทำให้เกลี่ยให้เรียบได้ยาก เกิดตุ่มเล็ก ๆ หากเกิดกรณีนี้ ให้เปิด ‘Stable smoothing’ ในตัวเลือกของเครื่องมือ smooth

:::

### รักษาขอบคม {#keep-sharp-edges}
เปิดการ snap จุดใหม่ให้ตรงกับขอบคมของ mesh เดิม อาจทำให้เกิดการบิดเบือน

## ![](/icons/dynamic.webp) ไดนามิกทอปอโลยี {#dynamic-topology}

![](/images/topology_dyntopo_menu.webp)
Multiresolution และ voxel remeshing เป็นวิธีควบคุมทอพอโลยีที่ใช้กันทั่วไปในอุตสาหกรรม แต่ทั้งสองแบบต้องคอยระวังไม่ให้โพลิกอนถูกยืดมากเกินไปหรือบีบแน่นเกินไป

Dynamic Topology เป็นวิธีทางเลือก ระหว่างที่คุณปั้น Nomad จะเพิ่มและลบโพลิกอนแบบปรับตามพื้นที่ระหว่างจังหวะแปรง ดังนั้นการแกะรายละเอียดเล็ก ๆ จะเพิ่มโพลิกอนเล็กจำนวนมากเฉพาะบริเวณที่ต้องการ และการเกลี่ยในส่วนอื่นจะลบโพลิกอนออก

โปรดทราบว่า dynamic topology จะใช้โพลิกอนแบบสามเหลี่ยมแทนควอด ซึ่งอาจดูรกเล็กน้อย แต่โดยทั่วไปแล้วควรเลิกมอง wireframe แล้วโฟกัสที่การปั้นให้สวยโดยไม่ต้องกังวลเรื่องทอพอโลยี จากนั้นค่อยใช้เครื่องมือ remesh อื่นของ Nomad เพื่อสร้าง quad mesh ที่สะอาดในภายหลัง

### เปิดใช้งาน {#enabled}
เปิด dynamic topology จะมีไอคอน DynTopo ใต้สไลเดอร์รัศมีและความแรงของแปรงเพื่อให้คุณสลับเปิด/ปิด Dyntopo ต่อเครื่องมือได้

### รายละเอียด {#dyn-detail}
ควบคุมปริมาณรายละเอียด พฤติกรรมจะเปลี่ยนไปตามตัวเลือก ‘Detail based on...’ ด้านล่าง

### รายละเอียดอ้างอิงจาก... {#detail-based-on}
| Method   | Description                                                     |
| :------: | :-------------------------------------------------------------: |
| Screen   | ระดับรายละเอียดขึ้นกับขนาดของอ็อบเจ็กต์บนหน้าจอ สไลเดอร์ detail ที่ 100% หรือมากกว่าจะให้รายละเอียดสูง ใช้สามเหลี่ยมเล็ก ๆ หรือ 1% จะให้รายละเอียดต่ำ ใช้สามเหลี่ยมใหญ่ |
| Radius   | รัศมีของเครื่องมือกำหนดปริมาณรายละเอียด ใช้รัศมีเล็กเพื่อรายละเอียดสูง รัศมีใหญ่เพื่อรายละเอียดต่ำ สไลเดอร์ detail จะเป็นตัวคูณของอัตราส่วนนี้ |
| Constant | สไลเดอร์ detail กำหนดปริมาณรายละเอียดโดยตรง ไม่ได้รับผลจากขนาดหน้าจอหรือขนาดเครื่องมือ |

::: tip TIP: โหมด Radius

เพื่อให้เข้าใจโหมด radius ได้ดีขึ้น ให้ลองเลื่อนสไลเดอร์ detail ด้วยนิ้วหนึ่ง แล้วเปลี่ยนรัศมีเครื่องมือด้วยอีกนิ้วหนึ่งพร้อมกัน คุณจะเห็นว่าทั้งสองอย่างเชื่อมโยงกันอย่างไร

:::

### ให้ความสำคัญกับ... {#prefer}
| Method  | Description       |
| :-----: | :---------------: |
| Speed   | เน้นประสิทธิภาพ |
| Quality | เน้นคุณภาพ       |

เมื่อเลือก `Quality` ความแตกต่างหลัก 2 อย่างคือ:
- การ refine จะทำก่อนการปั้น ดังนั้นคุณจะได้ artifact จากการ interpolate น้อยลงเมื่อเพนต์หรือปั้นรายละเอียดเล็กมาก
- การ refine จะทำต่อเนื่องจนกว่าจะลู่เข้าสู่ระดับรายละเอียดที่ถูกต้อง แทนที่จะเป็นแบบเพิ่มทีละนิด

ด้วยวิธีนี้ หากคุณปั้นรายละเอียดเล็กมากหรือปัดแปรงเร็ว ๆ ทอพอโลยีจะถูก refine ตามที่คาดหวังเสมอ


### ใช้แรงกดกับรัศมี {#use-pressure-on-radius}
ใช้ได้เฉพาะเมื่อเปิด `Radius` เมื่อเปิดแล้ว ระดับรายละเอียดจะสะท้อนขนาดแปรงเสมอ แม้ขนาดแปรงจะถูกปรับด้วยแรงกดปากกา

### ใช้สโตรกฟอลออฟ {#use-stroke-falloff}

นำเส้นโค้ง falloff ของแปรงและ alpha มาร่วมคำนวณ dyntopo ด้วย

ไม่ว่าคุณจะใช้ `Dynamic Topology` บน [Brush](#brush) หรือ [Global](#global) คุณสามารถเลือกโหมดการทำงานได้ดังนี้:

| Method         | Description                                                           |

การใช้ [Quadremesh tool](tools.md#quad-remesher) กับอ็อบเจ็กต์โพลิกอนสูงอาจใช้เวลาคำนวณนาน และให้ผลลัพธ์ที่ควบคุมได้ยาก การเตรียม mesh ล่วงหน้าด้วย [facegroups](tools.md#facegroup-1) และ decimate จะทำให้ Quadremesh ทำงานเร็วขึ้นมาก และให้การควบคุมทอพอโลยีได้มากขึ้น

* สร้าง facegroup ให้ mesh เพื่อกำหนดทิศทางควอดที่คุณต้องการ
* ใช้ Facegroup relax เพื่อให้ขอบ facegroup เรียบ
* Decimate เพื่อให้แน่ใจว่าไม่มีหน้าแคบหรือบิดเบี้ยวบนขอบ facegroup ในการตั้งค่า decimate ให้เปิด facegroup เพื่อให้ขอบสามเหลี่ยมตามขอบ facegroup อย่างแม่นยำ
* ในตัวเลือก Quadremesh ให้ปิด relax (เพราะคุณได้ relax mesh แล้ว) และคุณควรได้ผลลัพธ์ที่ดี

:::

#### ลดจำนวน {#decimate}
เริ่มการทำ decimate

ไอคอนข้างปุ่ม decimate ให้คุณสลับตัวเลือกที่มีผลต่อการ decimate ค่าเปอร์เซ็นต์บอกความแรงของตัวเลือกนั้น และสามารถตั้งค่าได้ในเมนูเฟืองขั้นสูง

* ![](/icons/palette.webp)  `Preserve Painting` - วางสามเหลี่ยมมากขึ้นในบริเวณที่มีรายละเอียดการเพนต์
* ![](/icons/triforce.webp) `Uniform Faces` - ให้ความสำคัญกับการสร้างสามเหลี่ยมขนาดใกล้เคียงกัน
* ![](/icons/hole.webp)  `Preserve Geometry Borders` - Decimate จะพยายามรักษาขอบใกล้เรขาคณิตเปิดและรูให้ไม่เปลี่ยนแปลง
* ![](/icons/facegroup.webp) `Preserve Facegroup Borders` - Decimate จะพยายามรักษาขอบ facegroup ให้ไม่เปลี่ยนแปลง
* ![](/icons/uv.webp) `Preserve UV Borders` - Decimate จะพยายามรักษาขอบ UV ให้ไม่เปลี่ยนแปลง

#### ![](/icons/cog.webp) เมนูลดจำนวนเฟือง {#decimate-gear-menu}
เมนูเฟืองมีตัวเลือกขั้นสูงดังนี้:
##### รักษางานเพนต์ {#preserve-painting}
ช่องติ๊กจะเปิด/ปิดโหมดนี้ ค่าเปอร์เซ็นต์กำหนดว่ารายละเอียดการเพนต์จะถูกเก็บไว้แม่นยำแค่ไหน ค่ายิ่งสูงยิ่งเก็บการเพนต์มาก ตั้งเป็น 0 หากคุณไม่สนใจการเพนต์


##### หน้าใบหน้าแบบสม่ำเสมอ {#uniform-faces}
ช่องติ๊กจะเปิด/ปิดโหมดนี้ ค่ายิ่งสูงจะยิ่งได้สามเหลี่ยมที่มีขนาดใกล้เคียงกัน

##### รักษาขอบ {#preserve-borders}
เปิดเพื่อหยุดไม่ให้ขอบถูก decimate สามารถเลือกน้ำหนักขอบสำหรับขอบ `Geometry`, `Face Group` หรือ `UV`

#### เป้าหมายจำนวนไตรแองเกิล {#target-triangles}
ตั้งค่าเป้าหมายจำนวนสามเหลี่ยม ค่าเริ่มต้นคือ 50% ปุ่ม percent/target จะสลับระหว่างการใช้เปอร์เซ็นต์หรือจำนวนโพลิกอนเป้าหมายแบบระบุจำนวน



### คลี่ UV - UVAtlas {#uv-unwrap-uvatlas}

::: tip UVs?
อุปมาอุปไมยที่ใช้บ่อยสำหรับ UV คือการห่อของขวัญ; วิธีที่ดีที่สุดในการตัดกระดาษห่อเพื่อคลุมวัตถุให้มิดโดยไม่ซ้อนทับกันคืออะไร?

UV ก็คล้ายกัน แต่แทนที่จะตัดกระดาษ คุณตัดตัววัตถุเอง ลองจินตนาการว่ารุ่นของคุณทำจากพลาสติกบาง ๆ คุณจะตัดมันอย่างไรเพื่อคลี่ให้นอนราบ เพนต์บนสภาพที่แบน แล้วประกอบกลับ?

ตอนนี้ลองจินตนาการว่าพื้นผิวของโมเดลทำจากผ้ายืด lycra คุณอาจยืดโมเดลให้แบน หรือตัด หรือทั้งสองอย่างผสมกัน แต่ถ้าคุณเพนต์ลายตารางหมากรุกตอนที่มันถูกแผ่ให้แบน ลายจะบิดเบี้ยวเมื่อประกอบกลับ วิธีไหนดีกว่า ระหว่างตัดมากแต่บิดเบือนน้อย หรือ ตัดน้อยแต่บิดเบือนมาก?

UV คือชุดคำสั่งบอกซอฟต์แวร์ 3D ว่าควร ‘ตัดและยืด’ วัตถุอย่างไรเมื่อใช้เท็กซ์เจอร์ เครื่องมือ UV Atlas โดยทั่วไปใช้แนวทาง ‘ตัดมาก บิดเบือนน้อย’


:::

::: tip UV กับ Nomad และแอปอื่น

โมเดลที่มีเท็กซ์เจอร์ส่วนใหญ่ที่คุณพบออนไลน์จะใช้ UV Nomad สามารถนำเข้าและแสดงผลได้ผ่านพาเนล [material](material#textures)

เมื่อสร้างโมเดลใน Nomad คุณสามารถเพนต์ลงบนวัตถุได้โดยไม่ต้องมี UV หากต้องการส่งออกไปยังแอปอื่น เช่น [Procreate](https://procreate.art/) คุณสามารถ ‘bake’ ข้อมูลสีของ Nomad ลงในเท็กซ์เจอร์ผ่าน UV ได้

:::

### เบก -> เทกซ์เจอร์ {#bake-texture}
![](/images/topology_bake_menu.webp)

การ bake เท็กซ์เจอร์จะสร้างเท็กซ์เจอร์โดยการ project อ็อบเจ็กต์อื่นที่มองเห็นในซีนลงใน UV ของอ็อบเจ็กต์ที่เลือก

เวิร์กโฟลว์ทั่วไปของการ bake มีดังนี้:
- คุณมี mesh ที่มีรายละเอียดและการเพนต์ละเอียด
- Clone มัน
- Decimate (ตั้ง `Preserve painting` เป็น 0)
- UV unwrap
- แล้วก็ Bake!

โดยค่าเริ่มต้น Nomad จะนำทุก mesh ที่มองเห็นในซีนมาคิดรวม  
คุณยังสามารถใช้โหมด Solo เพื่อซ่อน mesh อื่นส่วนใหญ่ได้อย่างรวดเร็ว  
หากไม่มีอ็อบเจ็กต์อื่นที่มองเห็นเลย มันจะใช้ทั้งซีนเป็นเป้าหมาย

ตอนนี้คุณควรมี mesh ความละเอียดต่ำที่ยังคงรักษาสีและรายละเอียดส่วนใหญ่ของอ็อบเจ็กต์เดิมไว้ได้

หลังการทำงาน สีของ vertex จะถูกย้ายไปยังเลเยอร์ใหม่ที่ถูกปิดการใช้งาน เพื่อไม่ให้รบกวนเท็กซ์เจอร์

#### จากตัวมันเอง {#tex-from-itself}
Bake ระดับ multiresolution สูงสุดลงสู่ระดับต่ำสุดของอ็อบเจ็กต์ปัจจุบัน วิธีนี้ตั้งค่าง่าย แต่บ่อยครั้งคุณอาจต้องการการควบคุมมากกว่านี้ ซึ่งตัวเลือกถัดไปจะมีประโยชน์กว่า

#### จากความละเอียดสูง () {#tex-from-high-res}
Bake จากอ็อบเจ็กต์อื่นที่มองเห็นในซีนลงสู่อ็อบเจ็กต์ที่เลือก ตัวเลขในวงเล็บบอกจำนวนอ็อบเจ็กต์อื่นที่มองเห็นซึ่งจะถูกใช้เป็น high-res targets และถูก bake ลงในอ็อบเจ็กต์ low-res ปัจจุบันที่มี UV อ็อบเจ็กต์อื่นไม่จำเป็นต้องมีรูปทรงหรือทอพอโลยีคล้ายกับอ็อบเจ็กต์ที่ถูก bake ทำให้รองรับเวิร์กโฟลว์ bake ที่ยืดหยุ่น

#### ความละเอียด {#tex-bake-resolution}
ความละเอียดของเท็กซ์เจอร์ที่ถูก bake เท็กซ์เจอร์ที่ bake จะเป็นสี่เหลี่ยมจัตุรัสเสมอ ดังนั้น 1024 จะสร้างภาพ 1024x1024

ปุ่มด้านล่างเป็นทางลัดสำหรับความละเอียดที่ใช้บ่อย สำหรับอ้างอิง 512x512 ถือว่าเล็ก เหมาะกับกราฟิกเว็บและเรขาคณิตง่าย ๆ ส่วน 4096x4096 (ย่อว่า 4k) เหมาะกับงานเรนเดอร์คุณภาพสูง

#### ![](/icons/cog.webp) เมนูเบกเฟือง {#tex-bake-gear-menu}
![](/images/topology_bake_gear_menu.webp)
เมนูเฟืองมีตัวเลือกขั้นสูงดังนี้:

##### นอร์มอล, ความหยาบ, ความเป็นโลหะ, สี, การเปล่งแสง, ความทึบแสง {#tex-normal-roughness-metalness-color-emissive-opacity}
ช่องติ๊กเหล่านี้กำหนดว่าจะ bake คุณสมบัติใดบ้าง แต่ละอย่างจะถูก bake ลงในแผนที่ (map) แยกกัน หลังจาก bake เสร็จ แผนที่เหล่านี้จะถูกเพิ่มเป็นเท็กซ์เจอร์ให้กับ material ของอ็อบเจ็กต์ปัจจุบัน

##### สำรองข้อมูล {#tex-backup}
เพื่อพรีวิวเท็กซ์เจอร์ที่ bake ข้อมูลการเพนต์ของอ็อบเจ็กต์ควรถูกปิด ตัวเลือกนี้จะย้ายข้อมูลการเพนต์ไปยังเลเยอร์ใหม่เป็นสำรอง เพื่อให้เปิด/ปิดได้ง่าย

#### รัศมีเคจ {#tex-cage-radius}
ปรับระยะห่างจากอ็อบเจ็กต์ที่ใช้ bake ที่จะส่งรังสีออกไปหาอ็อบเจ็กต์เป้าหมาย โดยค่าเริ่มต้นระยะนี้จะถูกตั้งให้ต่ำเพื่อหลีกเลี่ยง artifact แต่สามารถเพิ่มได้หากอ็อบเจ็กต์เป้าหมายอยู่ไกลจากอ็อบเจ็กต์ที่ใช้ bake

##### ออฟเซ็ตรังสี {#tex-ray-offset}
ปรับตำแหน่งเริ่มต้นของการคำนวณ bake บนอ็อบเจ็กต์ที่ใช้ bake โดยค่าเริ่มต้นจะเริ่มที่ระยะ 5% จากพื้นผิว ซึ่งช่วยหลีกเลี่ยง artifact ส่วนใหญ่ หากอ็อบเจ็กต์เป้าหมายอยู่ไกลจากอ็อบเจ็กต์ที่ใช้ bake ระยะ offset นี้อาจต้องเพิ่มขึ้น

![](/images/topology_reproject_menu.webp)

Project รายละเอียดการปั้น การเพนต์ เลเยอร์ เท็กซ์เจอร์ ลงเป็นค่าที่ระดับ vertex

สามารถมองว่าเป็นสิ่งตรงข้ามกับการ bake; ถ้า bake คือการถ่ายโอนคุณสมบัติของ vertex ไปยังเท็กซ์เจอร์ reproject คือการถ่ายโอนเท็กซ์เจอร์ (และคุณสมบัติอื่น ๆ) กลับลงสู่ vertices

::: tip
เมื่อใช้ `Bake to texture` หรือ `Reproject to vertex` ทั้งสีของ vertex และเท็กซ์เจอร์ของ material จะถูกนำมาคิดรวม
:::

#### จากตัวมันเอง {#vertex-from-itself}
แปลงเท็กซ์เจอร์จาก material เป็นค่าที่ระดับ vertex ปุ่มนี้จะใช้งานได้เฉพาะเมื่ออ็อบเจ็กต์มี UV และมีเท็กซ์เจอร์เปิดใช้งานใน material

::: tip TIP: การเพนต์เท็กซ์เจอร์
Nomad ไม่รองรับการเพนต์และแก้ไขเท็กซ์เจอร์โดยตรง แต่สามารถทำผลลัพธ์ที่ใกล้เคียงได้โดยการ project เท็กซ์เจอร์ -> ค่า vertex เพนต์บน vertices แล้วค่อย bake vertex -> เท็กซ์เจอร์:

1. ตั้งค่าอ็อบเจ็กต์โพลิกอนต่ำที่มี UV
1. โหลดเท็กซ์เจอร์เข้าไปใน material
1. Subdivide ให้มากพอสำหรับการเพนต์
1. ใช้ `Reproject to vertex` ในโหมด `From itself` ตอนนี้เท็กซ์เจอร์ถูกแปลงเป็นค่าที่ระดับ vertex แล้ว
1. เพนต์ เกลี่ย ป้าย ลายปั๊ม ทำการแก้ไขตามต้องการ
1. ใช้ `Bake to texture` ในโหมด `From itself` การแก้ไขเหล่านั้นจะถูกแปลงกลับเป็นเท็กซ์เจอร์
:::

#### จากความละเอียดสูง () {#vertex-from-high-res}
แปลงอ็อบเจ็กต์ที่มองเห็นทั้งหมดให้เป็นค่าที่ระดับ vertex บนอ็อบเจ็กต์ที่เลือก ตัวเลขบนปุ่มนี้บอกจำนวนอ็อบเจ็กต์ที่มองเห็น

::: tip
การ reproject อ็อบเจ็กต์อื่นไม่เพียงใช้ถ่ายโอนข้อมูลสีจากอ็อบเจ็กต์อื่นเท่านั้น แต่ยังใช้ project vertices ลงบนอ็อบเจ็กต์อื่นได้ เช่น project ผ้าพันแผลลงบนตัวละคร
:::

#### ![](/icons/cog.webp) เมนูรีโปรเจ็กต์เฟือง {#vertex-reproject-gear-menu}
เมนูเฟืองมีตัวเลือกขั้นสูงดังนี้:

#### เวอร์เท็กซ์, ความหยาบ, ความเป็นโลหะ, สี, ความทึบแสง, ความทึบแสง->มาสก์, มาสก์, เลเยอร์, กลุ่มหน้าใบหน้า {#vertex-vertices-roughness-metalness-color-opacity-opacity-mask-mask-layers-face-group}
ช่องติ๊กเหล่านี้กำหนดว่าคุณสมบัติใดจะถูก project ลงสู่อ็อบเจ็กต์ที่เลือก

#### ผ่อนคลาย {#vertex-relax}
mesh ที่เลือกสามารถถูกเกลี่ยหรือ relax รูปทรงได้ในระดับหนึ่งเพื่อให้เข้ากับเป้าหมาย reprojection ได้ดีขึ้น Smooth เหมาะกับ mesh โพลิกอนสูง Relax เหมาะกับ mesh โพลิกอนต่ำ Auto จะให้ Nomad เลือกวิธีที่เหมาะสมที่สุด

#### จำนวนรอบ {#vertex-iterations}
จำนวนครั้งที่ควรใช้การ relax ระหว่างการ reproject

#### รัศมีเคจ {#vertex-cage-radius}
ปรับระยะห่างจากอ็อบเจ็กต์ที่เลือกที่จะส่งรังสีออกไปหาอ็อบเจ็กต์เป้าหมาย โดยค่าเริ่มต้นระยะนี้จะถูกตั้งให้ต่ำเพื่อหลีกเลี่ยง artifact แต่สามารถเพิ่มได้หากอ็อบเจ็กต์เป้าหมายอยู่ไกลจากอ็อบเจ็กต์ที่ใช้ bake

#### ไบอัสดรังสี {#vertex-ray-bias}
ค่าต่ำจะเน้น project ไปยังจุดที่ใกล้ที่สุดบนพื้นผิวเป้าหมาย ค่าสูงจะเน้นจุดตัดตาม normal ของพื้นผิว

#### ออฟเซ็ตรังสี {#ray-vertex-offset}
ปรับตำแหน่งเริ่มต้นของการคำนวณ bake บนอ็อบเจ็กต์ที่เลือก โดยค่าเริ่มต้นจะเริ่มที่ระยะ 5% จากพื้นผิว ซึ่งช่วยหลีกเลี่ยง artifact บางประเภท หากอ็อบเจ็กต์เป้าหมายอยู่ไกลจากอ็อบเจ็กต์ที่เลือก ระยะ offset นี้อาจต้องเพิ่มขึ้น

### รู {#holes}
![](/images/topology_holes_menu.webp)

:::
