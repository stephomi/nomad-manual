# ![](/icons/open.webp) ไฟล์ {#files}

เมนูไฟล์ช่วยให้คุณบันทึกและโหลดโปรเจกต์ Nomad, นำเข้าและส่งออกโมเดล 3 มิติ และส่งออกภาพเรนเดอร์

![](/images/file_menu.webp)

## โปรเจกต์ {#project}
![](/images/file_project.webp)

จะแสดงภาพตัวอย่างของการบันทึกล่าสุดที่ด้านบนของเมนูนี้ การแตะที่ภาพตัวอย่างจะเปิดมินิบราวเซอร์ แตะสองครั้งที่โปรเจกต์อื่นเพื่อเปิดมินิเมนูสำหรับ เปิด, เพิ่ม, บันทึก, โคลน, เปลี่ยนชื่อ, ลบ โปรเจกต์นั้น

### ![](/icons/nomad.webp) พรีเซ็ต {#preset}
เข้าถึงชุดเดโมและชิ้นส่วนตัวละคร เลือกหนึ่งรายการ แล้วเลือกอีกครั้งเพื่อเลือก เปิด, เพิ่มเข้าในซีน หรือ โคลน รายการนั้นไปยังโฟลเดอร์โปรเจกต์ของคุณ

![](/images/file_preset_preview.webp)

### ![](/icons/save.webp) บันทึก {#save}
บันทึกโปรเจกต์ Nomad

### ![](/icons/save_as.webp) บันทึกเป็น... {#save-as}
แสดงบราวเซอร์โปรเจกต์เพื่อให้คุณบันทึกโปรเจกต์ Nomad ด้วยชื่อใหม่

### ![](/icons/pencil.webp) เปลี่ยนชื่อ {#rename}
แสดงกล่องข้อความเพื่อเปลี่ยนชื่อโปรเจกต์ปัจจุบัน

### ![](/icons/open.webp) เปิด... {#open}
แสดงบราวเซอร์โปรเจกต์เพื่อเปิดโปรเจกต์

### ![](/icons/add_file.webp) เพิ่มเข้าไปในซีน... {#add}
แสดงบราวเซอร์โปรเจกต์ เมื่อเลือกโปรเจกต์แล้ว เนื้อหาของมันจะถูกรวมเข้ากับซีนปัจจุบัน

### ![](/icons/trash.webp) ลบ... {#delete}
แสดงบราวเซอร์โปรเจกต์ โปรเจกต์ที่ถูกเลือกจะถูกลบออกจากระบบไฟล์

### ![](/icons/new_file.webp) สร้างใหม่ {#new}
เริ่มโปรเจกต์ใหม่ หากมีการเปลี่ยนแปลงที่ยังไม่บันทึก คุณจะถูกถามว่าต้องการบันทึกหรือไม่

### ![](/icons/autosave.webp) บันทึกอัตโนมัติ... {#auto-save}
เมนูสำหรับควบคุมตัวเลือกการบันทึกอัตโนมัติ

หากคุณเปิดใช้งานการบันทึกอัตโนมัติ คุณสามารถตั้งตัวจับเวลาที่จะให้ป๊อปอัปปรากฏเป็นระยะๆ  
เหตุผลที่ Nomad ไม่บันทึกเบื้องหลังคือไฟล์ 3 มิติอาจมีขนาดใหญ่ จึงอาจทำให้เกิดอาการหน่วงอย่างเห็นได้ชัดขณะปั้นโมเดล

นอกจากนี้ เพื่อหลีกเลี่ยงปัญหาหน่วยความจำไม่พอ ซีนมักจะถูกบีบอัดก่อนการบันทึก  
การบีบอัด/คลายการบีบอัดนี้จะทำให้การบันทึกช้าลงเช่นกัน

### หน้าต่างป๊อปอัปตัวจับเวลา {#timer-pop-up}
ความถี่ที่ป๊อปอัปตัวจับเวลาจะปรากฏ

### เวลาหน่วงป๊อปอัป {#popup-timeout}
เปิดใช้งานการหมดเวลาของป๊อปอัป

### ยกเลิกการบันทึกอัตโนมัติ {#discard-autosave}
หากมีไฟล์บันทึกอัตโนมัติสำหรับโปรเจกต์อยู่ มันจะถูกโหลดโดยอัตโนมัติแทนโปรเจกต์ต้นฉบับ หากไม่ต้องการ ให้กดปุ่มนี้เพื่อลบไฟล์บันทึกอัตโนมัติ การโหลดไฟล์จะกลับไปโหลดจากการบันทึกด้วยตนเองครั้งล่าสุดของโปรเจกต์นั้น


## นำเข้า {#import}

### ![](/icons/add_file.webp) นำเข้า {#import-button}
สำหรับการนำเข้าไฟล์ 3 มิติที่ไม่ใช่โปรเจกต์ Nomad

เมื่อคุณนำเข้าไฟล์ซีนภายนอกเข้าสู่ Nomad คุณสามารถเลือก *นำเข้า* หรือ *เพิ่ม* ได้

การเพิ่มไฟล์จะเป็นการเพิ่มอ็อบเจ็กต์เข้าไปในซีนปัจจุบัน  
การนำเข้าไฟล์จะสร้างโปรเจกต์ Nomad ใหม่ที่มีอ็อบเจ็กต์ใหม่อยู่ในนั้น

Nomad สามารถนำเข้าฟอร์แมตเหล่านี้:
- Nomad (.nom)
- glTF (.glb, .gltf)
- OBJ (.obj)
- STL (.stl)
- PLY (.ply)
- FBX (.fbx, ทดสอบ)

### ![](/icons/cog.webp) ขั้นสูง {#advanced}
แสดงตัวเลือกการนำเข้าขั้นสูง:

### โปรเจกต์/ glTF / OBJ / STL / FBX {#project-gltf-obj-stl-fbx}
#### คงโทโพโลยี {#keep-topology}
โดยปกติ Nomad จะพยายามแก้ไขเรขาคณิตที่มีปัญหาเมื่อโหลด เปิดใช้งานตัวเลือกนี้เพื่อหยุดไม่ให้ Nomad เรียงลำดับจุด/หน้าใหม่ ลบจุด/หน้าที่ซ้ำ และลบจุดที่ไม่ได้ใช้งาน

#### ข้ามเท็กซ์เจอร์ {#skip-textures}
ข้ามการโหลดเท็กซ์เจอร์สำหรับฟอร์แมตที่รองรับ เช่น glTF

### โปรเจกต์ / glTF {#project-gltf}
#### คงการตั้งค่า GUI {#keep-gui-settings}
เปิดใช้งานการบันทึกการตั้งค่า GUI และการตั้งค่าโปรเจกต์ภายในไฟล์ Nomad .nom หรือ glTF

### OBJ {#obj}
#### แยก OBJ ตามกลุ่ม {#split-obj-by-groups}
เปิดใช้งานการแยกกลุ่ม OBJ เป็นอ็อบเจ็กต์แยกกัน

#### โหมดสี {#color-space}
ตั้งโหมดสีที่ตีความจาก obj เป็น Linear, sRGB หรือ Auto

### PLY {#ply}
#### โหมดสี {#color-space-ply}
ตั้งโหมดสีที่ตีความจาก ply เป็น Linear, sRGB หรือ Auto


### FBX {#fbx}
#### โหมดสี {#color-space-fbx}
ตั้งโหมดสีที่ตีความจาก obj เป็น Linear, sRGB หรือ Auto

### ทั้งหมด/ที่มองเห็น/ที่เลือก {#allvisibleselected}
สถานะปุ่มที่ใช้งานจะกำหนดว่าอ็อบเจ็กต์ใดจะถูกส่งออก ตัวเลขถัดจากไอคอนบอกจำนวนอ็อบเจ็กต์ที่จะถูกส่งออกสำหรับตัวเลือกนั้น

### สีเวอร์เท็กซ์ {#vertex-colors}
ส่งออกสีของจุด หากฟอร์แมตไฟล์รองรับ

### การเพนต์แบบ PBR {#pbr-paint}
สีจุดแบบ PBR จะถูกส่งออกเป็นแอตทริบิวต์สีจุดชุดที่สอง  
ช่องต่างๆ จะถูกแพ็กดังนี้:

|           | ช่อง (Channel) |
| :-------: | :-------------: |
| Roughness | R               |
| Metalness | G               |
| Masking   | B               |


### เลเยอร์ {#layers}
เลเยอร์รองรับผ่าน glTF morph targets  
Nomad ยังส่งออกสี ความหยาบ (roughness) และความเป็นโลหะ (metalness) แยกตามเลเยอร์ แต่ซอฟต์แวร์อื่นมักจะไม่รองรับ

### การเพนต์เลเยอร์ {#layer-painting}
ส่งออกการเพนต์เลเยอร์ ซึ่งมักถูกเพิกเฉยโดยซอฟต์แวร์อื่น

### กลุ่มหน้า (Face Group) {#face-group}
ส่งออก facegroup การส่งออกอาจรบกวนการทำงานของซอฟต์แวร์อื่นได้ในบางครั้ง

### นอร์มัล {#normals}
ส่งออกข้อมูลนอร์มัล โปรดทราบว่า Nomad จะคำนวณนอร์มัลของตัวเองเสมอเมื่อมีการนำเข้าไฟล์ฟอร์แมตอื่น

### แทนเจนต์ {#tangents}
ส่งออกข้อมูลแทนเจนต์ ใช้เมื่อโมเดลมี normal map

### เท็กซ์เจอร์ {#textures}
หากมีการเพิ่มเท็กซ์เจอร์เข้าในวัสดุ เท็กซ์เจอร์เหล่านั้นจะถูกส่งออก โปรดทราบว่านี่จะไม่ใช่การ bake เท็กซ์เจอร์ การ bake ทำผ่านตัวเลือก bake ในเมนูโทโพโลยี

### ปุ่มส่งออก {#export-button}
คลิกปุ่มนี้เพื่อส่งออกเรขาคณิตโดยใช้การตั้งค่าที่เลือก

::: tip เคล็ดลับ: นำเข้า roughness และ metalness ไปยัง Blender

Blender สามารถนำเข้า glTF/glb ได้ แต่จะไม่เข้าใจแอตทริบิวต์สีจุดสำหรับ metalness และ roughness โดยอัตโนมัติ หากต้องการใช้ ให้สร้างโหนด Vertex Color ในตัวแก้ไขวัสดุ ตั้งค่าคุณสมบัติให้เป็นแอตทริบิวต์สีถัดไป (มักเป็น Col.001) เชื่อมต่อโหนด 'Separate XYZ' ส่ง X ไปที่ roughness และ Y ไปที่ Metallic

วิดีโอนี้แสดงขั้นตอน:

![](/videos/blender_pbr.mp4)

:::

## เรนเดอร์ {#render}

ส่งออกภาพที่เป็นผลรวมของการตั้งค่าทั้งหมดในโปรเจกต์ (ไฟ, วัสดุ, post processing ฯลฯ) 

![](/images/file_render.webp)
### พรีวิว {#preview}

ปุ่มพรีวิวขนาดเล็กถัดจากชื่อเมนูจะทำให้ทูลบาร์มืดลงเพื่อช่วยพรีวิวผลลัพธ์สุดท้าย

### พื้นหลังโปร่งใส {#transparent-background}
เปิดใช้งานช่องอัลฟาสำหรับภาพเรนเดอร์ มีประโยชน์สำหรับการนำภาพเรนเดอร์ไปผสมกับภาพอื่นในโปรแกรม 2 มิติ โปรดทราบว่าไม่รองรับความโปร่งใสแบบบางส่วน

### แสดงอินเทอร์เฟซ {#show-interface}
เปิดใช้งานการรวม UI ของ Nomad เข้าไปในภาพเรนเดอร์

### อัตราส่วนเรนเดอร์ {#render-ratio}
ตัวคูณสำหรับความละเอียดของภาพ

### ขนาดสุดท้าย {#final-size}
ความละเอียดที่จะใช้สำหรับการเรนเดอร์ เมื่อเลือก `Custom` สไลเดอร์ความกว้างและความสูงจะถูกเปิดใช้งาน

เมื่อเมนูไฟล์ทำงานอยู่ จะมีเส้นประซ้อนทับในวิวพอร์ตเพื่อบอกบริเวณเรนเดอร์หากไม่ตรงกับความละเอียดหน้าจอ (โปรดทราบว่าคุณต้องอยู่ในโหมดแนวนอนเพื่อให้ถูกต้อง)

### ส่งออก png {#export-png}
คลิกปุ่มนี้เพื่อเริ่มกระบวนการเรนเดอร์ เมื่อเสร็จแล้วคุณสามารถเลือกวิธีบันทึกหรือแชร์ภาพได้
