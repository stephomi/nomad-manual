# ![](/icons/camera.webp) カメラ {#camera}

このメニューでは、カメラの作成と編集、およびカメラとのインタラクション方法を制御できます。

![](/images/camera_overview2.webp)

Nomad のカメラには、次のような用途があります。

* 正確な角度からスカルプトするためのビューをセットアップする
* 写真用カメラのようにオブジェクトをフレーミングする
* シーンをナビゲートするための一人称視点カメラとして使う
* アイソメトリックゲームや工業系レンダリング向けの平行投影カメラとして使う

## カメラの操作 {#control}

### 回転 {#rotation}
背景上で指を「1本」ドラッグするとカメラを回転できます。  
モデル上でドラッグすると、代わりにスカルプト操作が開始されます。

::: tip 背景に触れられない場合でもカメラを回転できますか？
はい。画面に指を「2本」置いて（パン／ズームジェスチャーを始めるように）から、指を「1本」離してください。
:::

### フォーカス / リセット {#focus}
モデルを「ダブルタップ」すると、その位置にフォーカスします。  
背景を「ダブルタップ」すると、選択中のメッシュにカメラがフォーカスします。

### 平行移動 {#translation}
指を「2本」動かすことで、カメラをパンできます。

### ズーム {#zooming}
ピンチジェスチャーでズームイン／ズームアウトできます。

### ロール {#rolling}
指「2本」を回転させることでビューを「ロール」できます。
::: warning
このジェスチャーは `trackball` 回転モードでのみ利用できます。
:::

### デスクトップ操作 {#desktop}

デスクトップでは、Alt/Opt キーを使ってカメラを操作します。

* tip を空のスペースでドラッグ = カメラ回転
* alt+tip ドラッグ = パン
* alt+tip ドラッグ後に alt を離す = ズーム（zbrush と同じ）

ペンに 2 つ以上のボタンがある Wacom タブレットでは、Wacom の設定でペン先とボタンを次のように設定します。

* tip = 左クリック
* 下側ロッカー = 中クリック
* 上側ロッカー = 右クリック

この設定により、ペンだけでカメラを操作できます。

* 上側ロッカー + ホバー移動 = カメラ回転
* 下側ロッカー + ホバー移動 = パン

## カメラコントロール {#camera-controls}

![](/images/camera_list.webp)

### ビュー {#views}
`Add View` を使ってカメラのビューポイントを保存できます。  
ビュー名をクリックすると、そのビューが復元されます。

::: tip
保存されたビューには [Projection Type](#projection-type) の設定と [Reference image](background.md) も保存されます。  
異なる背景を持つ front/left/back のリファレンスビューを切り替えたい場合に便利です。
:::

| Action      | Icon                          | Description                                                                 |
| :---------: | :---------------------------: | :-------------------------------------------------------------------------: |
| Visibility  | ![](/icons/eye_open.webp)    | カメラの表示を切り替えます。非表示のカメラは前へ／次へボタンでスキップされます |
| Name        |                               | カメラを選択します                                                           |
| Image       | ![](/icons/image.webp)       | カメラにリンクされたリファレンス画像のサムネイル                            |
| Update View | ![](/icons/update_view.webp) | 現在のビューポイントで保存済みビューを更新します                           |
| Edit Name   | ![](/icons/pencil.webp)      | カメラ名を編集します                                                        |
| Delete      | ![](/icons/trash.webp)       | カメラを削除します                                                           |

### ![](/icons/tool_view.webp) ビューを追加 {#add}
現在のビューを元に新しいカメラを作成します。

### ![](/icons/camera.webp) アイコン {#icons-test}

ビューポート内にカメラアイコンを表示するかどうかを切り替えます。  
カメラが選択されている場合、そのアイコンは常に表示されます。

### 投影タイプ {#projection}
カメラの `Field of View`（FOV / 焦点距離）を変更できます。  
プロポーションを把握しやすくするため、スカルプト用途では低い FOV を使うことが推奨されます。  
`Orthographic` モードも使用でき、これは FOV が 0 に近い状態とほぼ同等です。

### 一人称視点 {#fps}
ピボットをスカルプトではなくカメラの位置に直接設定します。  
背景上で指をドラッグしてもカメラの位置は固定され、回転のみが変化します。これは一人称視点ゲームの挙動に似ています。単一オブジェクトではなく環境全体をスカルプトする際に便利です。

![](/images/camera_rotation_ortho_view.webp)

### 回転タイプ {#rotation-type}
デフォルトでは、カメラは `Turntable` 回転モードを使用します。  
これは自由度が 2 つだけで、直感的ですが、場合によってはより柔軟性が欲しくなることがあります。  
`Trackball` に切り替えると、ビューポート上で指「2本」を回転させることでビューを「ロール」できるようになります。デスクトップでは、一部のユーザーにとってより馴染みのある代替トラックボールモードもあります。

### 正投影スナップ {#orthographic}

有効にすると、キーボードで Shift を押しながらビューを回転させたとき、カメラが最も近い front/back/top/bottom/left/right ビューにスナップし、カメラが平行投影になります。  
また、ビューキューブをクリックして front/back/left/right/top/bottom にスナップした場合も、カメラは平行投影になります。

### ビューをリセット {#reset}

カメラを正面に移動し、シーン全体がビューに収まるようにします。

### ビューをスナップ {#snap}
最も近い front/back/left/right/top/bottom ビューにスナップします。  
すでにこれらのビューのいずれかにいる場合、もう一度クリックすると 180 度反転して反対側にスナップします。

### スピード {#speed}

カメラの動きが遅すぎる、または速すぎると感じる場合、`rotation`、`translation`、`zooming` の速度倍率を設定できます。  
非常に大きい、または非常に小さいスカルプトを扱う際に便利です。

### ピボット概要 {#pivot}

カメラを回転させると、小さなピンクの点が表示されます。これがカメラのピボットポイントです。  
ピボットがどこにあるかを理解しておくことは、カメラ操作で迷子になったりイライラしたりしないために非常に重要です。

デフォルトでは、ピボットは次の操作で更新されます。
- モデルをダブルタップ
- 背景をダブルタップ（新しいピボットはメッシュの中心になります）
- 画面に指を「2本」置く（パン／ズーム／ロール）と、指「2本」の中心にピボットが更新されます

### ピボットを更新... {#update-pivot}

ピボットの更新タイミングを、次のオプションでさらにカスタマイズできます。

* カメラの移動開始時
* 空中ピボットを許可
* ダブルタップ時
* スカルプト（ストローク）

::: tip
慣れてきたら、[Settings](settings.md) メニューでピンクの点（ヒント）を非表示にできます。
:::

### オブジェクトをダブルタップ {#dtap-object}
`Focus` が有効な場合、オブジェクトをダブルタップするとピボットがタップしたオブジェクトに移動します。

### 背景をダブルタップ {#dtap-tap-background}
有効な場合、ピボットを Selection、Scene のいずれか、またはその間のトグルに設定します。
